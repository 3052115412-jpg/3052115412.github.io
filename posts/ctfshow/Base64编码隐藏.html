HTTP头注入

攻击本质：应用程序未对用户输入的HTTP头部字段进行严格过滤或转义，导致攻击者可以通过构造恶意数据污染服务器响应头或执行非预期操作。

常见注入点：

  1. User - Agent（用户代理）
  2. Referer（来源地址）
  3. Cookie
  4. X - Forwarded - For（代理链）
  5. 自定义头字段（如 X - API - KEY）
  • 协议转换：https->http
  • 密码逻辑：虽然题目源码里有 Base64 字符串 Q1RGe2Vhc3lfYnFzZTYOfQ==，但实际上服务器最后只认 User-Agent，密码反而不是最重要的（或者说这是一个误导项，用来迷惑人的）。
  • 步骤： * proxy * intercept on * HTTP history找到，右键send to repeater， * User-Agent改为ctf-show-brower， * 最后一行加username=admin&password=CTF%7beasy_base64%7d * send
